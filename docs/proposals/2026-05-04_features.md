# PerfectlySnug — Data Modeling & Feature Set Spec
## 2026-05-04 · feature-set version `v1.0`

> Premise: per the 2026-05-04 context audit, the next controller iteration
> is **state-driven**, not time-driven. Inputs to the policy are body
> physics, room physics, movement physics, and a small set of recency
> counters. Time-of-night is not a feature — it is, at most, an offline
> evaluation slice. Override events are NOT features (they are labels);
> the controller never reads its own writes.

This spec defines:

1. raw signal inventory (HA / PG sources, units, freshness)
2. derived feature definitions (closed-form, exact windows)
3. normalization policy (per-feature)
4. `Snapshot` dataclass to drop into `ml/schema.py`
5. in-memory rolling buffer architecture for AppDaemon
6. missing-data / freshness behavior
7. feature-set versioning written to PG

The spec is tabular and type-hinted. It is intended to be implemented
as-is.

---

## 1. Raw signal inventory

All `<zone>` ∈ {`left`, `right`}. "HA" rows poll the Home Assistant state
machine; "PG" rows query Postgres `192.168.0.3:5432/sleepdata`.
"Controller" rows are owned by the AppDaemon process itself.

| signal_name | source (HA entity id OR PG column) | unit | sample_rate | max_staleness_seconds | NaN_policy |
|---|---|---|---|---|---|
| `body_left_f.<zone>` | HA `sensor.smart_topper_<zone>_side_body_sensor_left` | °F | 30 s (firmware push) | 120 | NaN if state ∈ {`unknown`, `unavailable`, `""`}; signal flagged stale; downstream feature flagged degraded. |
| `body_center_f.<zone>` | HA `sensor.smart_topper_<zone>_side_body_sensor_center` | °F | 30 s | 120 | same |
| `body_right_f.<zone>` | HA `sensor.smart_topper_<zone>_side_body_sensor_right` | °F | 30 s | 120 | same |
| `topper_ambient_f.<zone>` | HA `sensor.smart_topper_<zone>_side_ambient_temperature` | °F | 30 s | 120 | NaN as above. **Logged only — never used as room temp** (audit: 5–10 °F bias when blower idle). |
| `setpoint_f.<zone>` | HA `sensor.smart_topper_<zone>_side_temperature_setpoint` | °F | 30 s | 120 | NaN as above |
| `blower_pct.<zone>` | HA `sensor.smart_topper_<zone>_side_blower_output` | % (0–100) | 30 s | 120 | NaN as above |

| `pm_occupied_60s.<zone>` | PG `controller_pressure_movement.occupied` | bool | 60 s | 180 | NULL ⇒ fall back to live HA `bed_occupied` |
| `right_rail_engaged` | HA `input_boolean.snug_right_rail_engaged` | bool | event | 60 | unknown ⇒ `False`. IPC only — not a safety arm. |
| `current_setting.<zone>` | controller in-memory mirror of the last L1 it wrote, plus reconciliation against HA `bedtime_temp_dial` | int −10…0 (`left`); −10…+5 (`right`, capped 0 in policy) | event | n/a | always present; bootstrapped from HA on init. |

| `last_bed_onset_ts.<zone>` | controller; rising edge of `bed_occupied.<zone>` after ≥10 min vacancy | unix epoch s | event | n/a | None ⇒ "not in bed this session"; body sensors flagged invalid. |
| `bedjet_active` | controller; true while `body_left_f − topper_ambient_f > 5 °F AND body_trend_5min > 0.5 °F/15min` (heuristic — BedJet not in HA) | bool | derived 60 s | 300 | NaN inputs ⇒ `False`. |

## 2. Derived features (exact definitions)

Notation: `t` = current snapshot epoch seconds. `buf[s, w]` = ordered
samples of signal `s` whose timestamps satisfy `t − w ≤ ts ≤ t`, NaN
samples excluded. `slope(buf, unit)` = OLS regression of value on time
expressed in `unit per 15 min` (so all trend features share scale).
Counts use closed-open windows `[t − w, t)`. All derived features are
emitted per-zone unless explicitly shared.

### 2.1 Movement features

| feature | window | formula | unit | min samples |
|---|---|---|---|---|
| `movement_rms_5min.<zone>` | 5 min | `sqrt( mean( pm_abs_delta_sum_60s² for samples in window ) )` | %·sample | 3 of 5 minute-rows |
| `movement_rms_15min.<zone>` | 15 min | same, 15-row window | %·sample | 8 of 15 |
| `movement_variance_15min.<zone>` | 15 min | `var( pm_abs_delta_sum_60s )` (population variance, ddof=0) | (%·sample)² | 8 of 15 |
| `movement_density_15min.<zone>` | 15 min | `sum( pm_max_delta_60s ≥ 1.0 ) / minutes_with_data` | events/min | 8 of 15 |
| `movement_max_burst_15min.<zone>` | 15 min | `max( pm_max_delta_60s in window )` | % | 1 |

`movement_density_15min` matches the v6 `movement_density_15m` already
defined in `sql/v6_schema.sql` and `ml/v6/regime.py`; re-using the name
is intentional.

### 2.2 Body features

`body_avg_f.<zone>` = `mean({body_left_f, body_center_f, body_right_f})`
over the **non-NaN** subset; if all three are NaN, feature is NaN.

| feature | window | formula | unit | min samples |
|---|---|---|---|---|
| `body_avg_f.<zone>` | instant | mean of the 3 sensors | °F | 1 of 3 |
| `body_max_f.<zone>` | instant | max of the 3 sensors | °F | 1 of 3 |
| `body_min_f.<zone>` | instant | min of the 3 sensors | °F | 1 of 3 |
| `body_spread_f.<zone>` | instant | `body_max_f − body_min_f` | °F | 2 of 3 |
| `body_trend_5min.<zone>` | 5 min | `slope(buf[body_avg_f, 5min], °F per 15min)` | °F / 15min | 4 samples |
| `body_trend_15min.<zone>` | 15 min | `slope(buf[body_avg_f, 15min], °F per 15min)` | °F / 15min | 8 samples |
| `body_trend_30min.<zone>` | 30 min | `slope(buf[body_avg_f, 30min], °F per 15min)` | °F / 15min | 12 samples |

### 2.3 Room features

Shared (single sensor for the bedroom).

| feature | window | formula | unit |
|---|---|---|---|
| `room_temp_f` | instant | last good Aqara reading | °F |
| `room_trend_30min` | 30 min | `slope(buf[room_temp_f, 30min], °F per 15min)` | °F / 15min |

### 2.4 Recency / counter features

| feature | formula | unit |
|---|---|---|
| `seconds_since_presence_change.<zone>` | `t − ts_of_last_edge( bed_occupied.<zone> )`; `+inf` if no edge observed this session | s |
| `seconds_since_last_user_adjustment.<zone>` | `t − last_user_adjustment_ts.<zone>`; `+inf` if none | s |
| `seconds_since_bed_onset.<zone>` | `t − last_bed_onset_ts.<zone>`; `+inf` if not in bed | s |
| `controller_writes_last_30min.<zone>` | count of controller-initiated writes to `bedtime_temp_dial.<zone>` with `ts ∈ [t−1800, t)` | int |

### 2.5 Validity / gate flags

These are not features the model consumes directly; they gate which
features the policy is allowed to use. They are persisted to PG so
offline analysis can reproduce the gating decisions.

| flag | predicate | rationale |
|---|---|---|
| `body_sensor_validity.<zone>` | `False` if `seconds_since_bed_onset.<zone> < 600` (10 min post-bed-entry warm-up) **OR** `body_avg_f.<zone> < room_temp_f + 3.0` (mattress reading, not skin — audit #2.1, generalized from `RIGHT_BODY_SENSOR_VALID_DELTA_F=6.0` to 3.0 to capture the lower bound). True only when both conditions clear. | Body sensors read room+3-4 °F on empty/freshly-occupied bed. Block body-feedback features until they read skin. |
| `room_sensor_validity` | `False` if `room_temp_f` stale > 600 s **OR** outside 50–90 °F | Aqara dropouts must not drag the controller into wrong-band logic. |
| `movement_validity.<zone>` | `False` if no PG row in last 180 s **OR** `pm_sample_count_60s.<zone>` was 0 in ≥3 of last 5 minutes (sensor offline vs. legitimately still bed) | Distinguish "no movement" from "no data". |
| `controller_freeze_active.<zone>` | `True` if `seconds_since_last_user_adjustment.<zone> < 3600` | The 60-min override freeze is not a feature; it is a hard policy gate exposed as a flag. |
| `bedjet_suspected` | as defined for raw signal `bedjet_active` | Body-feedback features should not drive cooler dial when sheets are externally heated. |

## 3. Normalization policy

The controller is small (≪1 k parameters' worth of decisions), runs
on-device, and learns slowly across nights. Normalization choices
balance interpretability, robustness to user variance, and the need
to keep an **identical numerical recipe** between live inference and
offline training/eval.

| feature group | normalization | justification |
|---|---|---|
| Temperatures (`body_*`, `room_*`, `setpoint_*`, `topper_ambient_*`) | **Raw °F** | Physical units. Thresholds (e.g. body 80 °F target, room 68 °F reference) are physically meaningful and shared across users. Scaling would obscure debugging. |
| Temperature trends (`body_trend_*`, `room_trend_30min`) | **Raw °F per 15 min** | Same — magnitudes (e.g. ±0.2 °F/15min) are physically anchored. |
| Movement RMS / variance / density (`movement_rms_*`, `movement_variance_15min`, `movement_density_15min`) | **Per-user, per-zone z-score**, with running mean/std maintained over a 14-night sliding window keyed on `(occupied=True AND seconds_since_bed_onset > 1800)` rows. Persisted to PG `controller_movement_norm(user_zone, mean, std, n, updated_at)`. Bootstrap: until n ≥ 5 nights, use raw values and flag `feature_quality.movement = "warmup"`. | Population variance differs strongly between left and right zones (the user is a heavier sleeper than the wife) and between users in general. Z-score makes "elevated movement" mean the same thing across zones. **Crucial:** the `(mean, std)` pair must be persisted and versioned so offline replay reproduces the exact normalization the live controller used at decision time. |
| Movement max burst (`movement_max_burst_15min`) | Raw % | A single hard movement event has an absolute meaning (sheet displacement). Don't z-score one-shot extremes. |
| Counts (`controller_writes_last_30min`) | Raw int | Small bounded range (0–6). |
| Recency (`seconds_since_*`) | Raw seconds, with `clip(0, 86400)` and `+inf → 86400` mapping at policy boundary only (not in feature value). | Preserves the distinction between "5 min ago" and "never" until the very last step. |
| Dial state (`current_setting`, `bedtime_temp_dial`) | Raw int | The control axis. Never normalize. |
| Booleans / flags | Raw `0/1` for ML; `bool` in code | Self-evident. |

**Forbidden:** dataset-wide z-score across all users/sessions, MinMax to
[0, 1], or any normalization computed at training time and not
re-computable at inference. A normalization is only allowed if its
parameters are persisted in PG keyed on (user, zone, feature_set_version).

## 4. Snapshot dataclass

Drop-in for `ml/schema.py`. The existing `FEATURE_SCHEMA` /
`TARGET_SCHEMA` dicts and `INFLUXDB_MEASUREMENTS` block in that file
are obsolete (see audit §5: LightGBM/Influx era) and should be removed
as part of the same patch.

```python
"""ml/schema.py — feature-set v1.0 (2026-05-04)."""
from __future__ import annotations

from dataclasses import dataclass, field, asdict
from typing import Any, Optional

# Bumped on every change to (a) raw signal list, (b) derived feature
# formula, (c) normalization recipe, (d) Snapshot field set. Never
# decrement. Persisted to PG `controller_readings.feature_set_version`.
FEATURE_SET_VERSION = "1.0"

INF = float("inf")
ZONES = ("left", "right")


@dataclass
class ZoneSnapshot:
    """All per-zone raw + derived features at a single decision instant."""
    zone: str  # "left" | "right"

    # Raw temperatures (°F)
    body_left_f: Optional[float] = None
    body_center_f: Optional[float] = None
    body_right_f: Optional[float] = None
    topper_ambient_f: Optional[float] = None
    setpoint_f: Optional[float] = None
    blower_pct: Optional[float] = None

    # Dial state
    current_setting: int = 0                   # last value controller wrote
    bedtime_temp_dial: Optional[int] = None    # last value HA reports
    three_level_off: bool = True

    # Presence / pressure
    bed_occupied: bool = False
    bed_pressure_pct: Optional[float] = None

    # Derived body features (°F unless stated)
    body_avg_f: Optional[float] = None
    body_max_f: Optional[float] = None
    body_min_f: Optional[float] = None
    body_spread_f: Optional[float] = None
    body_trend_5min:  Optional[float] = None   # °F per 15 min
    body_trend_15min: Optional[float] = None   # °F per 15 min
    body_trend_30min: Optional[float] = None   # °F per 15 min

    # Derived movement features (z-scored unless raw)
    movement_rms_5min_z:        Optional[float] = None
    movement_rms_15min_z:       Optional[float] = None
    movement_variance_15min_z:  Optional[float] = None
    movement_density_15min:     Optional[float] = None  # events/min, raw
    movement_max_burst_15min:   Optional[float] = None  # %, raw
    # Raw movement (kept for offline analysis / re-normalization)
    movement_rms_5min_raw:       Optional[float] = None
    movement_rms_15min_raw:      Optional[float] = None
    movement_variance_15min_raw: Optional[float] = None

    # Recency / counters
    seconds_since_presence_change:    float = INF
    seconds_since_last_user_adjustment: float = INF
    seconds_since_bed_onset:          float = INF
    last_user_adjustment_delta:       int = 0
    controller_writes_last_30min:     int = 0

    # Validity flags (per-zone subset)
    body_sensor_validity: bool = False
    movement_validity:    bool = False
    controller_freeze_active: bool = False

    # Per-feature freshness (epoch-seconds age at snapshot time;
    # +inf if never observed). Mirrors raw_signal table 1:1 and is
    # the source of truth for `is_fresh()` (see §6).
    age_s: dict[str, float] = field(default_factory=dict)


@dataclass
class Snapshot:
    """Full controller-decision input. One row per 60-s tick per zone."""
    ts: float                                  # unix epoch seconds, UTC
    feature_set_version: str = FEATURE_SET_VERSION

    # Shared across zones
    room_temp_f: Optional[float] = None
    room_trend_30min: Optional[float] = None    # °F per 15 min
    room_sensor_validity: bool = False
    right_rail_engaged: bool = False
    bedjet_suspected: bool = False

    left:  ZoneSnapshot = field(default_factory=lambda: ZoneSnapshot(zone="left"))
    right: ZoneSnapshot = field(default_factory=lambda: ZoneSnapshot(zone="right"))

    # ── Constructors ──────────────────────────────────────────────
    @classmethod
    def from_ha_state(
        cls,
        *,
        ts: float,
        ha_get: "callable[[str], Any]",        # entity_id -> state object
        buffers: "RollingBuffers",             # see §5
        controller_state: "ControllerState",   # see §5
    ) -> "Snapshot":
        """Build a Snapshot from live HA + in-memory buffers.

        - `ha_get(entity_id)` returns either the raw state string or
          a structured object; coercion to float happens here.
        - `buffers` provides the 30-min ring buffers per signal.
        - `controller_state` provides counters and last_* timestamps.
        Pure function: no side effects, no I/O. The caller must have
        already polled HA and updated buffers for this tick.
        """
        ...

    # ── Serialization ─────────────────────────────────────────────
    def to_pg_row(self) -> dict[str, Any]:
        """Flatten to a single dict suitable for INSERT into
        controller_readings (one row per zone — caller picks the zone)
        or an extended snapshots table. Keys are stable across the
        feature_set_version lifetime."""
        ...

    def to_dict(self) -> dict[str, Any]:
        return asdict(self)
```

A constructor `ZoneSnapshot.from_ha_state(zone, ha_get, buffer, state)`
is the natural inner helper; the outer `Snapshot.from_ha_state` builds
both zones plus the shared fields.

## 5. Rolling buffer architecture

The AppDaemon controller maintains an in-memory buffer of the last
**30 minutes** of every numerically-typed raw signal listed in §1.
30 min covers the longest derived window (`body_trend_30min`,
`room_trend_30min`) with no padding logic.

### 5.1 Container

```python
from collections import deque
from dataclasses import dataclass

WINDOW_S = 30 * 60   # 30 min

@dataclass
class Sample:
    ts: float        # unix epoch seconds
    value: float     # NaN allowed; consumers strip on read

class RingBuffer:
    """Monotonic-ts deque, trimmed to WINDOW_S on every append/read."""
    def __init__(self, window_s: int = WINDOW_S):
        self._dq: deque[Sample] = deque()
        self._window_s = window_s

    def append(self, ts: float, value: float) -> None: ...
    def trim(self, now: float) -> None: ...
    def values_in(self, now: float, w_s: int) -> list[Sample]: ...
    def latest(self) -> Sample | None: ...
    def is_fresh(self, now: float, max_stale_s: float) -> bool: ...

class RollingBuffers:
    """One RingBuffer per (signal_name, zone_or_None)."""
    body_left_f:  dict[str, RingBuffer]   # keyed by zone
    body_center_f: dict[str, RingBuffer]
    body_right_f: dict[str, RingBuffer]
    setpoint_f:   dict[str, RingBuffer]
    blower_pct:   dict[str, RingBuffer]
    topper_ambient_f: dict[str, RingBuffer]
    bed_pressure_pct: dict[str, RingBuffer]
    pm_abs_delta_sum_60s: dict[str, RingBuffer]
    pm_max_delta_60s:     dict[str, RingBuffer]
    pm_sample_count_60s:  dict[str, RingBuffer]
    room_temp_f: RingBuffer
    # ... booleans use a separate small dict tracking last edge ts
```

### 5.2 Memory footprint estimate

Worst-case sample rates (capped at 1 Hz at the buffer layer; faster
sources are decimated to 1 Hz on insert because no derived feature
needs sub-second resolution):

| signal class | rate | samples in 30 min |
|---|---|---|
| Topper sensors (body, ambient, setpoint, blower) at 30 s | 0.033 Hz | 60 |
| Topper sensors decimated to 1 Hz cap if firmware ever pushes faster | 1 Hz | 1800 |
| Bed pressure (sub-second event stream, decimated to 1 Hz) | 1 Hz | 1800 |
| PG movement aggregates (60 s) | 1/60 Hz | 30 |
| Aqara room temp | ~1/120 Hz | 15 |

A `Sample` is 2 floats + Python overhead ≈ 80 B. Total channels:
- 3 body × 2 zones × 60 = 360 samples
- (ambient, setpoint, blower) × 2 zones × 60 = 360
- bed_pressure × 2 zones × 1800 = 3600
- pm_* × 3 metrics × 2 zones × 30 = 180
- room = 15
- Total ≈ **4 515 samples × 80 B ≈ 360 KB** resident.

Even with 2× headroom for deque internals, the buffer footprint is
well under **1 MB** — comfortable inside AppDaemon.

### 5.3 Persistence on restart (warm-start)

On `initialize()` the controller queries PG to backfill the 30-min
window so the first decision after a restart does not see degraded
trends. Two queries:

```sql
-- (a) body / setpoint / blower / topper-ambient — pulled from
-- controller_readings, which logs every 5-min cycle.
SELECT ts, zone,
       body_left_f, body_center_f, body_right_f,
       ambient_f       AS topper_ambient_f,
       setpoint_f, blower_pct
  FROM controller_readings
 WHERE ts >= NOW() - INTERVAL '30 minutes'
 ORDER BY ts;

-- (b) movement aggregates (60-s rows) — already in the right shape.
SELECT ts, zone,
       abs_delta_sum_60s AS pm_abs_delta_sum_60s,
       max_delta_60s     AS pm_max_delta_60s,
       sample_count      AS pm_sample_count_60s,
       occupied          AS pm_occupied_60s
  FROM controller_pressure_movement
 WHERE ts >= NOW() - INTERVAL '30 minutes'
 ORDER BY ts;
```

Room temp and bed-pressure live high-frequency streams are NOT
backfilled from PG (they aren't logged at sufficient resolution); the
controller marks the corresponding derived features `quality="warmup"`
for the first 15 min after restart.

Last-user-adjustment timestamp is recovered from the most-recent
`action='override'` row in `controller_readings` for each zone:

```sql
SELECT zone, MAX(ts) AS last_user_ts,
       (ARRAY_AGG(override_delta ORDER BY ts DESC))[1] AS last_delta
  FROM controller_readings
 WHERE action = 'override' AND zone IN ('left','right')
   AND ts >= NOW() - INTERVAL '24 hours'
 GROUP BY zone;
```

If PG is unreachable at boot, the controller starts with empty buffers
and emits all features with `quality="cold_start"` for the first
30 min, during which the policy is restricted to safe defaults
(setting=0, body-feedback disabled).

## 6. Missing data / freshness

Each signal `s` has an associated `max_staleness_s` from §1. Two
predicates are evaluated at every snapshot construction:

```python
def is_fresh(buf: RingBuffer, now: float, max_stale_s: float) -> bool:
    last = buf.latest()
    return last is not None and (now - last.ts) <= max_stale_s

def is_valid(value: float | None) -> bool:
    if value is None:
        return False
    if isinstance(value, float) and (value != value):  # NaN
        return False
    return True
```

A raw signal is **available** iff `is_fresh AND is_valid(latest)`.

A derived feature's `quality` is computed by combining the availability
of every input signal:

| derived-feature quality | meaning | policy effect |
|---|---|---|
| `ok` | all inputs available; min-sample count met | feature usable as-is |
| `warmup` | <14 nights of normalization data OR <15 min since cold start; feature numerically OK | feature usable but logged with warning; learner should down-weight |
| `degraded` | one or more inputs stale OR min-sample count not met; feature value still emitted from whatever data exists (e.g. trend on 5 of expected 12 samples) | state estimator must use the **degraded path**: trend features fall back to an EMA of available samples; body features fall back to the most-recent valid `body_avg_f`; room comp disabled if `room_temp_f` stale. |
| `unavailable` | zero usable inputs | feature value `None`; policy substitutes the safe-default branch (e.g. movement-driven proxy disabled, falls back to body-only). |

The `Snapshot` carries a parallel `feature_quality: dict[str, str]`
keyed by feature name, so the policy and the eval framework can both
inspect each feature's degradation state. Persisted to PG as JSONB
column `feature_quality` on the snapshot row.

**Hard rule:** a `degraded` body-feedback feature is fail-closed —
the controller acts as if `body_sensor_validity=False`, i.e. uses
room+setpoint physics only, no body-driven cool/warm correction.
Mirrors the existing `bodyFbFailClosed` patch (§2.1 of audit).

## 7. Versioning

`FEATURE_SET_VERSION = "1.0"` is a constant in `ml/schema.py` and is
written into Postgres on every snapshot row. Schema migration:

```sql
-- Round 1B v6 schema migration. Pure additive.
BEGIN;
ALTER TABLE controller_readings
    ADD COLUMN IF NOT EXISTS feature_set_version VARCHAR(16),
    ADD COLUMN IF NOT EXISTS feature_quality JSONB;

CREATE INDEX IF NOT EXISTS idx_controller_readings_fsv
    ON controller_readings (feature_set_version)
 WHERE feature_set_version IS NOT NULL;
COMMIT;
```

### Versioning rules

1. Bump `FEATURE_SET_VERSION` on **any** of:
   - addition / removal of a raw signal in §1,
   - change to a derived feature formula or window in §2,
   - change to a normalization recipe in §3 (including a new persisted
     `(mean, std)` snapshot for movement z-scores — those carry their
     own `norm_version` foreign key but bumping the feature set is
     required when the recipe itself changes),
   - change to the `Snapshot` field set in §4.
2. **Never** decrement; **never** reuse.
3. Use semver (`MAJOR.MINOR`):
   - MINOR: additive (new feature, old features unchanged).
   - MAJOR: breaking (feature removed, formula changed, normalization
     changed). Eval framework refuses cross-MAJOR comparisons by
     default; add `--allow-cross-major` only when the analyst can
     explain why the comparison is meaningful.
4. The eval framework (`tools/v6_eval.py`) filters
   `controller_readings` by `feature_set_version` for any
   apples-to-apples comparison. Aggregates that mix versions must
   carry an explicit `feature_set_version IN (...)` clause and emit
   a warning in the report header.
5. The persisted movement-normalization table also carries
   `feature_set_version` so a re-derivation of `movement_rms_*_z` from
   raw values is reproducible from PG alone.
