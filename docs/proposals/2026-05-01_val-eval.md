# v6 validation/evaluation framework — val-eval (2026-05-01)

This is the required gate for every v6 PerfectlySnug proposal. The executable implementation is `tools/v6_eval.py`; proposals pass by implementing `Policy.decide(state, history) -> int` and running through `run_eval()`.

## Non-negotiable leakage rules

1. **Temporal replay only:** at cycle `t`, a policy may use only `state[t]` plus `history[<t]` from the same zone-night.
2. **Forbidden fields:** `action`, `override_delta`, `notes`, `user_pref`, `revealed_pref`, future overrides, final-night labels/outcomes, next settings, and any feature computed with centered/future windows.
3. **Sleep stages:** use only a stage segment whose `start_ts <= t`. If Apple/Health arrives after the cycle in production, it is not available in honest replay unless the candidate explicitly models that reporting lag.
4. **No random KFold.** Readings inside a night are autocorrelated; override clusters are not independent.
5. **No L1-only RC modeling:** when RC/3-level mode is involved, use `L_active` (L1/L2/L3 selected by `run_progress`) when available. Historical windows without `run_progress` must be labeled as lower-confidence.
6. **Room sensor:** use external `room_temp_f`, not topper onboard `ambient_f`, for room/environment features.
7. **Drop synthetic off rows:** `running=off` blower zeros are integration artifacts and must not supervise blower/comfort.

`tools/v6_eval.py` enforces this by whitelisting state fields in `state_from_row()` and never passing override labels into the policy.

## Metrics

### Primary: MAE vs human override settings, per zone

For override rows in `controller_readings`:

- `human_setting = setting` because current v5 logs write the new manual value into `setting`.
- `controller_before = effective` if present, else `setting - override_delta`.
- `policy_error = abs(policy_decision_before_override - human_setting)`.
- Report: `n`, MAE, hit rate `abs(error) <= 1`, signed bias, and counts where candidate is strictly better/worse than observed v5.2.

Primary numbers are reported overall, by zone, and on final holdout nights.

### Secondary metrics

Implemented exactly in `add_comfort_proxies()`:

- **time-too-cold proxy:** minutes with `body_left_f < 76°F` and policy setting `<= -7`.
- **time-too-hot proxy:** minutes with `body_left_f > 84°F` on left or `>86°F` on right and policy setting `>= -5`.
- **spurious overrides predicted:** count of contiguous runs where `abs(policy_setting - observed_effective) >= 3` and no actual override row occurs at that cycle.
- **override-precluded count:** override events where `policy_error <= max(0, observed_v5_error - 2)`. This is intentionally strict: the proposal must be at least 2 L-steps closer than v5.2 to count as precluding the override.

### Right-zone comfort proxy (override-absence trap guard)

Right has only 7 overrides, so silence is not proof of comfort. For every right-zone row, compute a 0..1 discomfort score:

```
right_comfort_proxy =
  0.35 * body_out_of_range
+ 0.25 * body_30m_sd_excess
+ 0.20 * stage_bad
+ 0.20 * restlessness
```

Definitions:

- `body_out_of_range = clip(max((body_left_f - 86)/6, (73 - body_left_f)/5), 0, 1)`.
- `body_30m_sd_excess = clip((rolling_30m_sd(body_left_f) - 1.2)/2.0, 0, 1)`.
- `stage_bad = 1` for `awake`, `inbed`, or `unknown`, else `0`.
- `restlessness = clip(abs(diff(bed_right_calibrated_pressure_pct))/8, 0, 1)` using PG 5-min pressure; when high-resolution movement is available, candidates may add it, but must compute it trailing-only.

Report mean, p90, and minutes with score `>=0.5`.

## Replay protocol

1. Load `controller_readings` for both zones from PG.
2. Assign sleep nights using production convention: timestamps before 18:00 belong to the prior sleep night.
3. For each `(zone, night)`, sort by `ts` and iterate forward.
4. Build a leakage-safe `state` dict for the current cycle.
5. Call `policy.decide(state, history)`; clamp to integer `[-10, 0]`.
6. Append only the current state and decision to `history` after the decision.
7. Score override rows and proxy rows after replay.

### Counterfactual body dynamics for “what if”

The evaluator includes a clearly labeled simple assumption, not a causal claim:

```
steady_state_surface_delta_f = -0.03 * (policy_blower_pct - observed_blower_pct)
cf_delta[t] = cf_delta[t-1] + (1 - exp(-dt/45min)) * (steady_state - cf_delta[t-1])
```

Thus, reducing blower by 50 points warms the surface proxy toward `+1.5°F` with a 45-minute time constant. This is used for directional case-test pass/fail and should be replaced by a fitted model once controlled body-in-bed data exists.

### Ground truth weighting

- **Left:** manual overrides are primary comfort signals with weight 1 in primary MAE.
- **Right:** primary MAE uses right overrides, but right-zone selection must also improve or not worsen the composite right comfort proxy because override count is sample-starved.
- **Silent minutes:** not primary labels. They are only secondary/proxy evidence.

## Split strategy

- **Walk-forward by zone-night:** expanding training nights, next chronological night as validation.
- **Final test:** most recent 3 nights per zone are held out from model fitting/tuning and reported separately.
- **Per-zone splits:** left and right are split independently because right is sample-starved and physiologically different.
- **No random KFold:** any random row-level split leaks same-night thermal state, override freezes, and sleep-stage context.

## Statistical power and claim rule

Current data in the executable run: 61 overrides total (54 left, 7 right), 25 zone-nights with overrides.

Baseline variability from `tools/v6_eval.py`:

- Overall baseline MAE SD: `1.296` L-steps; approximate 80% power / alpha 0.05 MDE: `0.465` MAE steps.
- Left: n=54, SD `1.355`, MDE `0.516` MAE steps.
- Right: n=7, SD `0.756`, MDE `0.800` MAE steps, but this is unstable; right must rely on proxies until more overrides accrue.

**Required improvement claim:** a candidate may claim improvement only if all are true:

1. MAE improvement vs v5.2 observed baseline is at least `max(0.5 L-step, estimated MDE)` overall and not directionally harmful in either zone.
2. Zone-night block bootstrap 95% CI for `MAE(candidate) - MAE(v5.2)` is entirely `< 0`.
3. No required case test fails.
4. Right comfort proxy p90 and minutes `>=0.5` do not increase unless right override MAE materially improves and the change is clinically justified.

Bootstrap is by full zone-night block, never by row.

## Required case tests

### A — 2026-04-30→05-01 LEFT 01:37–02:05 too-cold cluster

Context: user repeatedly warmed left from controller path `-10 -> -9 -> -8 -> -6 -> -4 -> -3` while body-left was ~75.4–76.5°F.

Pass if, over `2026-05-01 01:37:00–02:05:59 America/New_York`:

- median predicted setting `>= -5`, and
- at least 20 minutes predicted `>= -6`, and
- median counterfactual surface delta `>= +1.0°F` vs observed.

### B — 2026-04-30→05-01 RIGHT 03:25 under-cooled override

Context: right manual override `-4 -> -5` at room `68.27°F`, `body_left=73.13`, `body_center=76.95`. Below-72 room warming would contradict this signal.

Pass if, over `03:10–03:40`:

- setting `<= -5` for at least 15 minutes, and
- median setting over `03:20–03:31` is `<= -5`.

### C — 2026-04-30 morning “cold mid-night, slightly warm in morning”

Context: left override at `04:27` warmed to `-2`; later `06:56` cooled to `-4`.

Pass if:

- median setting during `04:15–04:40` is `>= -3`, and
- median setting during `06:40–07:10` is `<= -4`.

## Output contract

`tools/v6_eval.py` emits JSON with these top-level fields:

- `policy_name`
- `generated_at`
- `primary`: overall/by-zone/final-test override MAE, hit rate, bias, observed v5.2 comparator
- `secondary`: time-too-cold/hot, spurious predicted override runs, override-precluded count/rate
- `right_comfort_proxy`: mean/p90/minutes high and exact definition
- `splits`: number of walk-forward folds and final test nights
- `power`: bootstrap CIs, MDE estimates, claim rule
- `cases`: A/B/C pass/fail plus numeric details and trajectory rows
- `rows`: readings/replayed/override counts

Run:

```bash
cd PerfectlySnug
.venv/bin/python tools/v6_eval.py --policy baseline --out ml/state/v6_eval_baseline.json
```

Custom policy:

```bash
.venv/bin/python tools/v6_eval.py --policy my_module:MyPolicy --json
```
